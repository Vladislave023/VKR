# VKR — прототип подсистемы валидации входных данных для электронной библиотеки

## О проекте

Данный проект представляет собой прототип пользовательской подсистемы информационной системы поддержки сотрудников Отдела электронной библиотеки Научной библиотеки ДВФУ.

Основная задача подсистемы — обеспечить **корректный ввод метаданных** при подаче выпускной квалификационной работы и **предварительную валидацию загружаемого файла** до этапа проверки сотрудником библиотеки.

Проект разработан в рамках выпускной квалификационной работы по теме:

**«Разработка прототипа информационной системы поддержки для сотрудников Отдела электронной библиотеки Научной библиотеки ДВФУ. Подсистема валидации входных данных»**

---

## Назначение системы

Подсистема предназначена для решения следующих проблем текущего процесса ввода данных:

- пользователь может ввести некорректное ФИО;
- год работы может быть введён в неверном формате;
- название работы может содержать запрещённые символы или некорректный регистр;
- в качестве файла может быть загружен не PDF;
- отсутствует жёсткий контроль обязательных полей;
- часть данных сотрудникам библиотеки приходится исправлять вручную.

Подсистема переносит основную проверку на этап **ввода пользователем**, тем самым снижая количество ошибок и облегчая работу сотрудников.

---

## Основной функционал

### Пользовательская часть
- вход в систему;
- подача заявки на загрузку ВКР;
- заполнение метаданных работы;
- выбор учебных данных из справочников;
- загрузка **одного PDF-файла**;
- просмотр списка своих заявок;
- просмотр карточки отдельной заявки;
- просмотр статуса заявки;
- просмотр комментария проверяющего.

### Валидация данных
- проверка ФИО автора;
- проверка ФИО руководителя;
- запрет инициалов;
- проверка использования только кириллицы в ФИО;
- проверка года;
- проверка количества страниц;
- проверка названия работы;
- запрет кавычек в названии;
- контроль капса в текстовых полях;
- проверка расширения и типа файла;
- ограничение размера файла до 100 МБ;
- контроль выбора направления в зависимости от уровня образования и школы;
- контроль выбора кафедры в зависимости от школы.

### Интерфейс
- клиентская валидация в реальном времени;
- подсветка некорректных полей;
- текстовые сообщения об ошибках возле поля;
- серверная валидация на стороне Django;
- AJAX-отправка формы без полной перезагрузки страницы.
